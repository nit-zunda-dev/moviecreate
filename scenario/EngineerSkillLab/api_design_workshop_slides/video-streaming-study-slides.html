API設計ワークショップ

動画配信APIをRESTで設計する

いいね・コメント・チャンネル登録を設計する4問

シナリオの前提

  • 未ログインでも動画視聴・動画詳細の取得は可能（public）
  • いいね・コメント投稿・チャンネル登録はログイン必須（bearer）
  • 動画配下の子リソース（likes/comments）と channels 配下の subscriptions を使う

エンドポイントの全体図

公開取得：GET /videos/{id}

  • 用途：動画のメタ情報と再生用URLを取得
  • 認証：public（Authorization なし）
  • 成功：200 OK + JSON
  • ヘッダ：Accept: application/json

いいね：POST /videos/{id}/likes

  • 用途：動画への「いいね」関係を作成
  • 認証：bearer
  • ヘッダ：Authorization と Content-Type: application/json
  • 成功：201 Created + JSON（別解: PUT + 200）

コメント投稿とチャンネル登録

ワークショップ回答例（4本）

用途MethodPathAuthStatus
作品詳細 GET /videos/{id} public 200
いいね追加 POST /videos/{id}/likes bearer 201
コメント投稿 POST /videos/{id}/comments bearer 201
チャンネル登録 POST /channels/{id}/subscriptions bearer 201

おつかれさまなのだ！

公開リソースと個人リソースを先に分離すると、
メソッド・パス・ヘッダーの整合が取りやすくなる。

高評価・チャンネル登録よろしくなのだ！
