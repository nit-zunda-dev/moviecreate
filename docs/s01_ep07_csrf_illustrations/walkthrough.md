# CSRF解説スライドの図解表現（HTML/CSS版）について

ご要望にお応えし、画像ファイルの代わりに **HTMLとCSS（Flexbox、グラデーション、アニメーションなど）** を活用して、最大限グラフィカルで直感的な表現を作り込みました。

## 更新内容

1. **共通のスタイリング追加（リッチUIコンポーネント）**
   - グラスモーフィズム（すりガラスのような半透明効果）とグラデーションを取り入れた立体的なパネルデザイン (`.rich-box`) を導入しました。
   - トランジション（ホバー時に浮かび上がる）や、CSSキーフレームを用いたアニメーション（ゆっくり浮かぶ、通信の矢印が流れる、警告が点滅する等）を追加し、静的な図よりも動きのある分かりやすい表現にしました。
   - 状態に合わせたテーマカラー（`theme-ok`（緑）、`theme-danger`（赤）、`theme-accent`（紫））を設定しました。

2. **スライドごとの図解の再構成**
   
   - **スライド2（CSRFインシデントの流れ）**
     被害者、攻撃者サイト、正規アプリを横に並べ、各要素間に「データが流れるアニメーション付きの矢印」を配置しました。ログイン状態から攻撃をトリガーし、意図しないリクエストが送られる様子を視覚的に表現しています。
   
   - **スライド5（XSSとCSRFの比較）**
     画面を左右に分割し、「ブラウザで爆発するスクリプト（XSS）」と「手紙（リクエスト）を意図せず送らされる（CSRF）」という対比を、絵文字アイコンと大きく強調したテキストボックスで表現しました。
   
   - **スライド6（同期トークン対策の仕組み）**
     トークン（鍵）を「バッジ」のようなUIで可視化しました。正規ページからサーバーへ正しいトークンが送られて照合成功する流れと、攻撃者サイトからのリクエストが「トークンを持たないために拒否される」様子をブロックの配置で明確に示しています。

## ご確認方法
ブラウザで再度 `s01-ep07-csrf-study-slides.html` を開いていただき、各スライドの表現をご確認ください。
画像生成を使用しなくても、見栄えの良いプロフェッショナルなスライドとして機能するように調整しております。
