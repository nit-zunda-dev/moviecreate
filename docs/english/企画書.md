# 企画書：「エンジニアの現場あるあるから学ぶ英会話」

## 基本情報

| 項目 | 内容 |
|------|------|
| シリーズタイトル | エンジニアの現場あるあるから学ぶ英会話 |
| サブタイトル案 | ずんだもんとめたんの現場Englishラボ |
| ターゲット | IT業界の英語に苦手意識がある初級〜中級エンジニア／就活・転職準備中の学習者 |
| 媒体 | YouTube（通常動画＋ショート切り抜き） |
| 更新頻度 | 週1本（目標） |
| 1話あたりの尺 | 6〜10分（通常版）／60秒以内（ショート版） |
| 形式 | 立ち絵アニメ＋会話劇＋フレーズ解説（VOICEVOX使用） |

---

## 企画コンセプト

「英語そのもの」ではなく、**エンジニアの現場で本当に飛び交う英語表現**を、ストーリーの中で体験しながら覚えるシリーズ。

ずんだもん（生徒役）が現場で困る側、四国めたん（先生役）が翻訳と意図の解説を担当することで、以下を同時に達成する。

- 現場の温度感に沿った自然な英語の理解
- 仕事で使える短い返答フレーズの定着
- 「聞き取れない・返せない」不安の軽減

---

## キャラクター設計

### ずんだもん（生徒役）
- 英語に対して苦手意識がある若手エンジニア
- 技術はできるが、英語会議で固まりがち
- 視聴者の「それ言いたかった」を代弁する
- 口癖：「〜なのだ！」

### 四国めたん（先生役）
- 実務経験豊富なバイリンガル寄りエンジニア
- 言い回しのニュアンス差を短く説明できる
- 「丁寧すぎる英語」「直接すぎる英語」を場面に応じて調整する

---

## 各話構成（テンプレート）

1. **現場あるある導入（60〜90秒）**  
   スクラム、障害対応、レビュー、顧客MTGなどのシーンで会話が発生。

2. **詰まりポイント可視化（30秒）**  
   ずんだもんが聞き取れない／返答に迷う。

3. **英会話解説パート（2〜3分）**  
   めたんが「直訳」「実務での意味」「使える返答」をセットで解説。

4. **言い換え練習パート（1〜2分）**  
   丁寧版・カジュアル版・緊急時版を比較。

5. **実戦リプレイ（60〜90秒）**  
   同じ場面を英語で言い直し、改善を体験。

6. **今日の3フレーズ復習（30秒）**  
   テロップで即復習、次回テーマ予告。

## 学習設計（教育面の差別化）

### 1. 現場文脈ファースト
- 文法解説を最小化し、実務で必要な発話を優先
- 1話につき「持ち帰り3フレーズ」に絞る

### 2. ニュアンス比較
- 同じ意味でも丁寧さ・緊急度・責任範囲で使い分ける
- 「言える」だけでなく「角が立たない」言い方まで扱う

### 3. リピート設計
- 冒頭で前回フレーズを再登場させ、忘却を防止
- ショート動画で1フレーズ1テーマに再編集

## 演出・制作方針

### ビジュアル
- 会議画面、チャット画面、PR画面などを背景に使用
- 重要フレーズは英語＋日本語＋使用シーンを同時表示
- 聞き取りづらい箇所は波形演出や強調テロップで補助

### 音声
- ずんだもん：VOICEVOX「ずんだもん」
- 四国めたん：VOICEVOX「四国めたん」
- 英語例文は聞き取り優先で速度を段階化（等速→やや速い）

### テロップ
- 「直訳」「意図」「返答例」を色分け表示
- 動画後半に復習カード形式のテロップを固定配置

---

## 投稿・運用方針

### 投稿導線
- 通常動画：週1本（学習本編）
- ショート：本編から3本切り出し（フレーズ復習）

### サムネイル方針
- 左側：ずんだもんの困り顔、右側：めたんの解説顔
- 大文字キーワード例：「会議で詰む英語」「その返し失礼かも」

### コミュニティ施策
- コメント募集：「次に扱ってほしい現場シーン」
- 固定コメントに「今日の3フレーズ」を記載

## 想定リスクと対策

| リスク | 対策 |
|--------|------|
| 英語が難しすぎて離脱される | 1話3フレーズ上限、復習を毎回固定化 |
| 現場設定が狭くマンネリ化する | 開発・運用・マネジメントの場面を循環させる |
| 発音への指摘が増える | 概要欄に「実務フレーズ理解重視」の方針を明記 |
| 学習チャンネル化しすぎて娯楽性低下 | あるある劇パート比率を全体の50%以上で維持 |
